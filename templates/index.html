GET /health

Health check endpoint to verify API status and model availability.

Response:

{
  "status": "healthy",
  "message": "API is running",
  "model_loaded": true
}
POST /predict

Predict sales for a single record.

Request Body:

{
  "Store_id": "S001",
  "Store_Type": "A",
  "Location_Type": "Urban",
  "Region_Code": 1,
  "Date": "2024-01-15",
  "Holiday": 0,
  "Discount": 1
}

Response:

{
  "success": true,
  "prediction": {
    "sales": 1234.56,
    "store_id": "S001",
    "date": "2024-01-15"
  },
  "input_data": { ... }
}
POST /predict/batch

Predict sales for multiple records at once.

Request Body:

{
  "data": [
    {
      "Store_id": "S001",
      "Store_Type": "A",
      "Location_Type": "Urban",
      "Region_Code": 1,
      "Date": "2024-01-15",
      "Holiday": 0,
      "Discount": 1
    },
    {
      "Store_id": "S002",
      "Store_Type": "B",
      "Location_Type": "Rural",
      "Region_Code": 2,
      "Date": "2024-01-15",
      "Holiday": 0,
      "Discount": 0
    }
  ]
}

Response:

{
  "success": true,
  "count": 2,
  "predictions": [
    {
      "sales": 1234.56,
      "store_id": "S001",
      "date": "2024-01-15"
    },
    {
      "sales": 987.65,
      "store_id": "S002",
      "date": "2024-01-15"
    }
  ]
}
POST /predict/csv

Upload a CSV file and get predictions for all records.

Request: Form-data with file upload (key: "file")

CSV Format:

Column Type Description
Store_id string Store identifier
Store_Type string Type of store (A, B, C, etc.)
Location_Type string Urban, Rural, etc.
Region_Code integer Numeric region code
Date date YYYY-MM-DD format
Holiday integer 0 or 1
Discount integer/string 0/1 or Yes/No

ðŸ“¦ Example Usage

Python (requests)

import requests
import json

# Single prediction
url = "http://localhost:5000/predict"
data = {
    "Store_id": "S001",
    "Store_Type": "A",
    "Location_Type": "Urban",
    "Region_Code": 1,
    "Date": "2024-01-15",
    "Holiday": 0,
    "Discount": 1
}

response = requests.post(url, json=data)
print(response.json())
